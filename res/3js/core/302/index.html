  • 정보

    • size

  • forEach - each는 jQuery로 $.each(collection, callback) 사용

[2, 5, 9].forEach(function(element, index, array) {
console.log(index + ":" + element);
/**
* 0:2
* 1:5
* 2:9
*/
});
  • filter - array에 바탕은 배열. index, array는 생략 가능.
var filtered = [12, 5, 8, 130, 44].filter(function(element, index, array) {
	return (element >= 10);
});

console.log(filtered); // [ 12, 130, 44 ]
  • map - array에 바탕은 배열. index, array는 생략 가능.
var square = [1, 2, 3, 4, 5].map(function(element, index, array) {
	return element * element;
});

console.log(square); // [ 1, 4, 9, 16, 25 ]
  • reduce
var total = [0, 1, 2, 3].reduce(function(previousValue, currentValue, index, array) {
	return previousValue + currentValue;
});

console.log(total); // 6
  • every, some - 결과는 boolean. array에 바탕은 배열. index, array는 생략 가능.
var passed = [12, 54, 18, 130, 44].every(function(element, index, array) {
	return (element >= 10);
});

console.log(passed); // true
passed = [2, 5, 8, 1, 4].some(function(element, index, array) {
	return (element >= 10);
});

console.log(passed); // false

let 변수는 this로 참조가 불가하다 !!

조심하자 !! let일 경우 에러가 발생한다.

ES5는

  1. 함수 선언문 설정
  2. 변수 이름을 바인딩, 변수 이름은 undefined

ES6에서 let은 호이스팅이 되지 않음 !!

즉, let변수 앞에서는 사용이 불가능하다. !!

let 변수는 변수를 선언한 이후 인식 할 수 있다.

const 변수는 바꿀수 없지만, Object 프로퍼티를 바꿀수 있음

const는 배열의 엘리멘트를 바꿀수도 있음. !!

화살표 함수

  • prototype과 constructor가 없음. prototype에 메소드를 확장 할 수 없으며, new로 인스턴스를 생성 할 수 없다.
  • arguments를 사용 할 수 없음.
  • 대신 rest 파라미터를 사용함.
  • 화살표 함수는 this는 없다. 단 window (글로벌오브젝트)를 참조한다.
