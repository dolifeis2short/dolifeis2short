# 자바 8

  • 함수형 인터페이스
    • 1개의 추상 메소드를 갖는 인터페이스
    • Java8 부터 인터페이스는 기본 구현체를 포함한 디폴트 메서드 (default method) 를 포함
    • 여러 개의 디폴트 메서드가 있더라도 추상 메서드가 오직 하나면 함수형 인터페이스

참조 (https://ko.myservername.com/interface-enhancements-java-8-java-functional-interface)

@FunctionalInterface
interface CustomInterface<T> {
// abstract method 오직 하나
T myCall();

    // default method 는 존재해도 상관없음
    default void printDefault() {
        System.out.println("Hello Default");
    }

    // static method 는 존재해도 상관없음
    static void printStatic() {
        System.out.println("Hello Static");
    }
}

함수형 인터페이스	Descripter	Method
Predicate	      T -> boolean	boolean test(T t)
Consumer	      T -> void	      void accept(T t)
Supplier	      () -> T	        T get()
Function<T, R>	T -> R	        R apply(T t)
Comparator	    (T, T) -> int	  int compare(T o1, T o2)
Runnable	      () -> void	    void run()
Callable	      () -> T	        V call()

  • 메서드 참조

    • Static 메소드 레퍼런스
    • Instance 메소드 레퍼런스
    • Constructor 메소드 레퍼런스 (ex, System.out::println)
    • 람다 표현식 화살표 함수를 의미

  • 스트림 API

    • intermediate operations return Stream
    • terminal operations return 특정한 type의 값
    • 순환, 필터링, 매핑, 중복제거, 조합

  • 날짜와 시간 API

    • java.time LIB 제공
    • Java8 이전 Date, Calendar 그리고 Timestamp, SimpleDateFormat 클래스 등 날짜와 시간을 다루는 API 존재 참조 날짜와 시간 API (https://wickso.me/java/java-8-date-time/)

  • 인터페이스 개선 Java 8 릴리스는 인터페이스에 정적 및 기본 메소드를 도입하거나 허용

  • Optional

    • Java Optional 클래스는 Java 8에서 추가되었으며 자바의 고질적인 문제인 NullpointerException 문제를 해결
    • Optional opt = Optional.of("saelobi"); Optional 메서드를 이용하면 다음과 같은 if를 이용한 null값 체크를 대체할 수 있음

  • CompletableFuture

    • 각각을 비동기로 실행하는 함수
    • CompletableFuture를 이용해서 별도의 Thread에 작업을 지시한 후 완료될때까지 다른 작업을 동시에 실행

  • 람다와 함수형 프로그래밍
