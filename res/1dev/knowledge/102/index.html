# 함수형 프로그래밍

# 1. 프로그래밍 종류

* 절차적 
* 객체지향적 - 객체와 데이터가 유착되어 응집도가 높은 패키지 형태
* 함수형 - 데이터를 숨길 필요가 없고, 작은 자료형을 유지

# 2. 개발 방식

간단한 함수들로 작업을 분리
3) 흐름 체인으로 데이터를 처리
4) 리액티브 패러다임을 실현하며, 이벤트 중심 코드로 복잡성을 줄인다.
- 함수형 언어는 결국, 분해와 합성을 통한 상호 작용을 통한 연산 작용

리액티브는 관찰이 가능해야 한다.

# 3. 개발 특징

1) 스트림을 통과한 데이터는 불변성 특징, 원천 데이터는 수정되지 않고 새롭게 생성된 데이터 
2) 스트림은 객체에 특화되었며, unboxing이 않일어나게 하기 위해서 interface를 사용 !!
3) 스트림은 한번 사용하고 나면, 다시 사용할 수 없다. 
4) 스트림 중 void를 호출하면, 전체 스트림 데이터를 소모하고 종료된다. (최종연산)
