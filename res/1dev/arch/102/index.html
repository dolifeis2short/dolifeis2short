# 보안이론

# 1. 정보보호의 3요소

  • 기밀성
    • 스누핑, 트래픽 분석
    • 인가된 사용자만이 정보에 접근할 수 있음
    • 비인가 사용자에게 정보가 노출되지 않음(보안등급)
  • 무결성
    • 변경, 가장, 재연, 부연
    • 인가된 사용자만이 권한에 따라 정보를 변경할 수 있음
    • 비인가 사용자의 정보 변경을 금지함(해쉬함수)
  • 가용성
    • DoS
    • 인가된 사용자가 시스템 자원을 필요로 할때 접근 가능함

# 2. 암호화 시스템

  • 양방향 (암호화, 복호화)
    • 대칭키
      • 스트림 암호 : 비트 또는 비트 쿢음 단위로 XOR
      • 블록 암호 : 크기를 블록 단위로 나눠 암호화, DES, 3DES, AES
    • 비대칭키 : RSA, DH
    • 기밀성 재공 (인증, 부인방지 X)
  • 일방향(암호화)
    • MDC(Hash), MAC(Hash+ 대칭키, 전자서명(Hash+비대칭키)
    • 인증, 서명 암호화 수행 (인증, 부인방지 기능 O)

# 3. 보안 취약점

  • 정보보호 공격유형
    • 변조 : 원래의 데이터를 조작
    • 가로채기 : 네트워크 상에서 전송되는 데이터 공격
    • 차단 : 정상적인 서비스를 방해하는 행위
    • 위조 : 송신되는 메시지를 변조하여 사용자를 속임
  • 공격 방법
    • XSS(Cross Site Scripting)
      • 실행 가능한 스크립트를 재전송하는 공격 기법
    • CSRF(Cross Site Request Forgery)
      • 사용자 권한을 탈취하여 웹 사이트를 요청

# 4. 보안 방지

  • 지식기반 인증
    • 패스워드 (무차별공격, 사전공격, 트로이목마, 스니핑 방지)
  • 소유기반 인증
    • 스마트카드, OTP
  • 존재 기반 인증
    • 손바닥, 손, 홍재
  • 행당기반 인증
정보접근 방법 
식별 : 사람이나 유일성을 확인 하는 절차
인증(Authentication) : log-on 
인가(Authorizationn) : 권한
