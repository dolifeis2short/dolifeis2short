# 2. 객체지향 설계 5대 원칙 ( SOLID )

  • 단일 책임 원칙(SRP)
    • 한가지 기능의 변경이 다른 곳에 영향을 주어서는 안된다.
  • 개방 / 폐쇄 원칙(OCP)
    • 개체는 확장 가능성을 열어두고, 수정 가능성은 닫아야 한다. (Extend)
  • 리스코프 치환 원칙(LSP)
    • 개체의 추상화 -어떤 타입에서 파생된 객체가 있다면 이 타입을 사용하는 코드는 변경하지 않아야한다.(abstract)
  • 인터페이스 분리 원칙(ISP)
    • 외부 인터페이스를 통해서 객체끼리 소통한다.
  • 의존성 역전 원칙(DIP)
    • 상위 수준 모듈은 하위 수준 모듈에 의존해서는 안된다. 이 둘읕 추상화에 의존해야 한다.
DRY  규칙
- 반복하지 마라. !! (Do not Repeat Yourself)

# 3. 객체지향 4대 원칙

  • 캡슐화 (Encapsulation) - 정보 은닉
  • 다형성 (Polymorphism) - 사용자 편의
  • 상속 (Inheritance) - 재사용
  • 추상화(Abstraction) - 모델링 (상속을 통한 추상화, 인터페이스를 통한 추상화, 다형성을 추상화)

# 4. UML

  • Generalization (object -> extends)
  • Realization (interface -> implementation)
  • Dependency (Setter, return Object)
    • 매서드 내에서 대상 클래스를 생성, 리턴 받아서 사용하는 것
    • 해당 클래스와 관계를 유지하지 않음
  • Association ()
    • 다른 객체의 참조를 가지는 필드
  • Aggregation / Composition
    • 생명주기가 Object 종속
